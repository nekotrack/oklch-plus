## はじめに

Oklab/Oklch は、知覚均等性を目指して設計された色空間です。しかし「知覚的均等性を目指した」ことと「達成している」ことは別の話です。

Oklch は Oklab の極座標表現です。L（明度）は両者で共通で、Oklab の直交座標 (a, b) が Oklch では C（彩度 = √(a²+b²)）と h（色相角 = atan2(b, a)）に対応します。同じ知覚空間を座標系の違いで表したものであり、C 軸が明示的に分離されているため、彩度方向の補正を直接設計できます。

色差とは、２色の色覚的な「違い（=距離）」を数値化したものです。値が大きいほど「違う」と感じる色の組み合わせになります。この色差をどう計算するかによって、モデルの精度が変わります。

CIEDE2000 といった精密な色差モデルと比較したとき、Oklab/Oklch 空間の色差（補間する二点間のユークリッド距離）には改善余地があります。本稿では、その改善を試みた過程と、COMBVD データセット（3,813ペア）を用いた評価結果を記録します。

結論として：**Oklab/Oklch 空間に 3パラメータの変換を加えることで、〜17の経験定数を持つ CIEDE2000 と同等以上の精度を達成しました。**

---

## 1. なぜ Oklab/Oklch の色差に改善余地があるのか

Oklch の L 軸は cube root（3乗根）で圧縮されており、明度方向の知覚非線形性はある程度補正されています。しかし **C（彩度）軸は色差計算において（形式上）線形距離として扱われています。**

一般的な知覚の圧縮特性から類推すると、低彩度域では彩度の差に敏感で、高彩度域では鈍感になると考えられます。sRGB のガンマ補正が輝度軸の非線形性を補正するのと同じ構造で、彩度軸にも圧縮が必要と考えました。

```
Oklab の ΔE = √(ΔL² + Δa² + Δb²)
             ← C 軸（= √(a²+b²)）は線形距離として計算
```

実際、COMBVD（CIE が整備した色差評価データセット。6つの独立した実験系・3,813ペアを収録）で計算すると：

STRESS（Standardized Residual Sum of Squares）は、色差モデルの予測値と心理物理実験の評価値（人間が実際に感じた「違い」の大きさ）の乖離を数値化した指標です。0 が完全一致・値が小さいほど正確なモデルを意味します。

| モデル | COMBVD STRESS ↓ |
|--------|----------------|
| Oklab（ベースライン） | 47.35 |
| CIEDE2000（参考） | 29.18 |

Oklab と CIEDE2000 の間には 18点ポイント以上の差があります。

CIEDE2000 は CIE が 2001 年に標準化した色差式で、約 17 の経験定数を持ちます。明度・彩度・色相の各軸に重み関数を持ち、現在も色差評価の事実上の標準として広く使われています。

## 2. 失敗の記録：色相変調による補正は汎化しなかった

最初に試みたのは、色相（H）方向への変調を加える補正です（HGamma と命名）。MacAdam 楕円（1942年・識別閾値実験データ）を Oklab 平面に投影すると、色相方向に応じて誤差のパターンが観察されます。これをもとに、色相角に依存した楕円補正を試みました。

**実際に今回試した HGamma は汎化しませんでした。**

MacAdam 楕円は識別閾値（最小弁別差）の実験データです。これを超閾値評価データセット COMBVD に適用した結果、全 6 サブデータセットで悪化しました：

| データセット | Oklab | HGamma 適用後 | 判定 |
|------------|-------|--------------|------|
| ALL（3,813ペア） | 47.35 | 64.27 | ❌ 大幅悪化 |
| BFD-P(D65) | 51.45 | 63.92 | ❌ 悪化 |
| RIT-DuPont | 31.76 | 80.06 | ❌ 大幅悪化 |

この結果を受け、色相依存の補正ではなく **C 軸の全方位的な圧縮** を試すことにしました。

なお HGamma の検討は識別閾値データを中心に行っており、COMBVD への適用が困難と判断した段階で一旦方針を転換したものです。色相方向の補正可能性を否定するものではありません。

## 3. C 軸圧縮の効果

最もシンプルな C 軸圧縮は冪乗変換（Power compression）です：

```
C' = C^γ
```

γ=0.675 で COMBVD 全体を最適化すると、STRESS は 47.35 → 32.10 まで改善しました。**1パラメータで STRESS を 32% 削減できる** というのは、C 軸線形近似が主要な誤差源であることの傍証です。

---

## 4. なぜ Naka-Rushton か

冪乗変換の次に検討したのが **Naka-Rushton 関数** です：

```
C' = C^n / (C^n + σ^n)
```

**中 研一（Ken-Ichi Naka）** と **William A.H. Rushton** が 1966 年に発表した研究に基づく関数で、元は網膜の光受容体（錐体・桿体）の光応答モデルです。弱い刺激には敏感に反応し、強い刺激では飽和していく S 字型の特性を持ちます。

冪乗変換との形状上の違いは **飽和特性** にあります：

```
冪乗  C' = C^0.675    → 高彩度域でも C' は単調増加し続ける
NR    C' = C^n/(C^n+σ^n) → C ≫ σ で 1 に収束（飽和）
```

σ パラメータが「感度の切り替わり点」を制御するため、データに応じた適応的なフィッティングが可能です。

---

## 5. 3パラメータで CIEDE2000 を超える

NR 関数による C 軸圧縮に、L 軸の冪乗補正（L' = L^α）を加えた 3パラメータモデルが **Oklch+** です：

```
L' = L^α
C' = C^n / (C^n + σ^n)
h' = h                          (unchanged; hue weight = C′ implicitly)
a' = C' cos(h),  b' = C' sin(h)
ΔE = √(ΔL'² + Δa'² + Δb'²)   ← 変換後の Oklab ユークリッド距離
```

改善の主要因は C 軸の NR 圧縮です。Section 3 の冪乗圧縮（1パラメータ）だけで STRESS 47.35 → 32.10（−15.25）の改善が得られたように、C 軸の線形性解消が最も効果的な介入点です。L 軸補正単独では STRESS 40.70（−6.65）にとどまります。L・C 両軸を同時最適化すると STRESS 29.10 まで到達します。

scipy（Python 科学計算ライブラリのひとつ）の Nelder-Mead 法で 5 つの初期点から最適化を行い、以下のパラメータに収束しました：

```
α = 0.73,  n = 0.87,  σ = 0.34
COMBVD STRESS = 29.10
```

性能比較には HELMLAB を含めています。色差モデル HELMLAB は Oklab を拡張した手法で、COMBVD に 72 パラメータをフィットさせ STRESS 22.48 を達成しています（[GitHub: Grkmyldz148/helmlab](https://github.com/Grkmyldz148/helmlab)）。

| モデル | STRESS ↓ | パラメータ数 |
|--------|---------|------------|
| Oklab | 47.35 | 0(追加前) |
| Oklch γ=0.675 | 32.10 | +1(追加数) |
| **Oklch+（α=0.73, n=0.87, σ=0.34）** | **29.10** | **+3(追加数)** |
| CIEDE2000 | 29.18 | 〜17(総パラメータ数) |
| HELMLAB v20b | 22.48 | 72(総パラメータ数) |

**Oklabから+3パラメータで CIEDE2000（〜17パラメータ）を超えました。**

HELMLAB との差（29.10 vs 22.48）は残ります。ただし HELMLAB は COMBVD 3,813ペアに 72パラメータをフィットしており（72/3,813 ≒ 1/53 の比率）、軽度の過適合が報告されています（公式サイト・arXiv:2602.23010）。独立データでの検証は要確認です。

### サブデータセット別スコア

COMBVD は 6 つの独立した実験データセットから構成されています。全サブセットでの改善は汎化の証拠になります。

| モデル | ALL | D65 | M | C | WITT | RIT | LEEDS |
|--------|-----|-----|---|---|------|-----|-------|
| Oklab | 47.35 | 51.45 | 41.69 | 42.22 | 45.01 | 31.76 | 45.15 |
| **Oklch+** | **29.10** | **23.95** | **28.11** | **34.29** | **24.27** | **25.16** | **34.05** |
| Δ | −18.26 | −27.49 | −13.58 | −7.93 | −20.74 | −6.60 | −11.10 |

全 6 サブデータセットで改善しています。改善幅は D65（BFD-P 実験系、全ペアの 53%）で最大（−27.49）です。

## 7. Gamut Mapping — Layer 3

Oklch+ による色差補正（Layer 2）は補間の品質を改善しますが、グラデーション経路上に **sRGB 色域（ディスプレイが表現できる色の範囲）を超えた色が生成される** という実用上の問題があります。つまりソフトウェア上で数値は指定できても、sRGB ディスプレイでは正確に表示できません。この現象については Oklchも同様の課題を内包しています。

ブラウザや CSS の標準的な処理は、この問題を「事後クリッピング」で対処します。つまり計算結果が色域外に出た後、特定チャンネルを強制的に 0〜1 に切り詰める処理です。この事後処理には問題があります：**クリッピング時に RGB チャンネルの比率が崩れ、輝度階調が不連続に変化する。**

輝度のコントラストは、色覚多様性のあるユーザーにとって色そのものより信頼性が高いとされています。仮に事後クリッピングで輝度構造が壊れた場合、色相ではなく輝度で情報を読み取るユーザーへの影響が相対的に大きくなると推定されます（実証的な確認は行っていません）。

これに対し、**事前に C 軸を出力デバイスの色域上限（C_max）以内に滑らかに圧縮**することで、CSS・ブラウザに渡る前の段階で色域問題を解消できます。実装は 2 モードです：

```
Conservative モード: C_max(L)     — 全色相共通の C 上限（安全な包絡線）
                                     色域内の高彩度色も若干圧縮されるため
                                     低彩度寄りのシフトが生じる
Full モード:        C_max(L, H)   — 色相ごとの正確な C 上限
                                     （5° 解像度・2D キャッシュ、計算コストは軽微）
```

Conservative モードは全色相を単一の L 依存関数でクランプするシンプルな実装です。色域外への漏れを防ぐ一方、色域内の色も圧縮されます。Full モードは色相ごとに上限を計算するため精度が高く、色域内の色を大きく変化させません。

Oklch+ 補間（Layer 2）→ Gamut Mapping（Layer 3）という順序で適用することで、知覚的に正しい補間を輝度構造を保ったまま色域内に収めるパイプラインを構成しています。

この Layer 3 により、事後クリッピングが実装されていない表示環境においても色域問題を解消できるポテンシャルを保有します。

## 8. 前稿との接続：C 軸パラメータの Hue 変調への転用

前稿「[OKLCH 極座標補間の特異点を「滑らかに」解消する](https://zenn.dev/nekotrack/articles/7c3487c5a9375a)」で使用した Hue 変調の重み関数は：

```javascript
w_h(C) = C^0.87 / (C^0.87 + 0.34^0.87)
```

n=0.87, σ=0.34 という値が、本稿の Oklch+ の C 軸最適化で得られたパラメータです。最適化の順序は **C 軸色差（COMBVD 評価）→ Hue 変調（補間経路の安定化）** であり、色差改善のために決めたパラメータを Hue 変調に転用したものです。

「なぜ Hue 変調にこのパラメータを使うのか」への答えは：**知覚的に均等な彩度スケールで色相への信頼度を制御するから** です。C=σ=0.34 付近で信頼度が 0.5 になる点は、Oklch の彩度スケールにおける知覚的な「中彩度」に対応しており、この点を境に色相の影響を段階的に落とすことが色かぶり解消に効いています。

この Hue 変調は、無彩色端点への補間だけでなく、**Blue→Yellow 等の有彩色間で補間経路の中間に低彩度域を通過するケース**でも同様に機能します。モダンブラウザでも残るこの問題に対して、同一のメカニズムで連続的に対処できる点が実用上の主要な価値です。

## 9. おわりに

3パラメータで CIEDE2000 を超えるという結果は、Oklch の C 軸が線形距離として扱われているという事実から来ています。Naka-Rushton 関数を選んだ主な理由は、飽和特性という関数の形状が彩度の知覚特性と合致すると考えたためです。

Oklch+ は平均的には CIEDE2000 に近い予測性能を持つモデルと考えていますが、パラメータの定義域端点（C→0、L→0、L→1、および C→C_max(L,H) 付近）での挙動確認が残っており、現時点では暫定的な評価です。C の上限は sRGB 色域の形状に依存するため、色相・明度ごとに異なる境界近傍での挙動は独立した検証が必要です。

残る課題：

- **Hue 軸の非線形性** — 今回の補正は L・C 軸のみで、Hue 方向の知覚的歪みは未対応です。HELMLAB が STRESS 22.48 を達成していることから考えると、検証すべき特徴量が、まだ複数残っている可能性もあり得ます。
- **ディスプレイ上での画像品質検証** — 心理物理実験の集計値（COMBVD）での改善が、実際の表示環境でどの程度体感できるかは別途確認が必要です。
