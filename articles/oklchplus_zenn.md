## はじめに

Oklab は、知覚的均等性を目指して設計された色空間です。しかし「知覚的均等性を目指した」ことと「達成している」ことは別の話です。

CIEDE2000 といった精密な色差モデルと比較したとき、Oklab の色差（ユークリッド距離）には改善余地があります。本稿では、その改善を試みた過程と、COMBVD データセット（3,813ペア）を用いた評価結果を記録します。

結論から言うと：**Oklab 空間に 3パラメータの変換を加えることで、〜17の経験定数を持つ CIEDE2000 と同等以上の精度を達成しました。**

---

## 1. なぜ Oklab の色差に改善余地があるのか

Oklab の L 軸は cube root（3乗根）で圧縮されており、明度方向の知覚非線形性はある程度補正されています。しかし **C（彩度）軸は色差計算において線形距離として扱われています。**

人間の色知覚は、低彩度域では彩度の差に敏感で、高彩度域では鈍感です。sRGB のガンマ補正が輝度軸の非線形性を補正するのと同じ構造で、彩度軸にも圧縮が必要です。

```
Oklab の ΔE = √(ΔL² + Δa² + Δb²)
             ← C 軸（= √(a²+b²)）は線形距離として計算
```

実際、COMBVD（CIE Combined Corrected Vision Dataset）で計算すると：

| モデル | COMBVD STRESS ↓ |
|--------|----------------|
| Oklab（ベースライン） | 47.35 |
| CIEDE2000（参考） | 29.18 |

STRESS とは「モデルの色差予測と人間の視覚評価の乖離」を示す指標です（0 が完全一致、小さいほど良い）。Oklab と CIEDE2000 の間には 18点以上の差があります。

---

## 2. 失敗の記録：色相変調による補正は汎化しなかった

最初に試みたのは、色相（H）方向への変調を加える補正です（HGamma と命名）。MacAdam 楕円を Oklab 平面に投影すると、色相方向に応じて誤差のパターンが観察されます。これをもとに、色相角に依存した楕円補正を試みました。

**ここで「閾値」と「超閾値」の違いを整理しておきます。**

| 種別 | 意味 | 実験例 |
|------|------|--------|
| **閾値（threshold）** | ギリギリ区別できる最小の色差 | MacAdam 楕円 |
| **超閾値（suprathreshold）** | 明確に「違う」と感じる色差 | COMBVD |

MacAdam 楕円は閾値実験です。わずかに色が異なる 2 点を繰り返し比較して「見分けられる最小差」を測ります。一方 COMBVD は超閾値実験で、「明らかに異なる 2 色がどれくらい違うか」という評価です。知覚の伝達関数がそもそも異なるため、閾値で有効な補正が超閾値で機能するとは限りません。

**実際に HGamma は汎化しませんでした。**

MacAdam 閾値実験から導いたパラメータを COMBVD（超閾値）に適用すると：

| データセット | Oklab | HGamma 適用後 | 判定 |
|------------|-------|--------------|------|
| ALL（3,813ペア） | 47.35 | 64.27 | ❌ 大幅悪化 |
| BFD-P(D65) | 51.45 | 63.92 | ❌ 悪化 |
| RIT-DuPont | 31.76 | 80.06 | ❌ 大幅悪化 |

全 6 サブデータセットで悪化しました。閾値と超閾値は別現象であるという、色科学における重要な注意点を実験的に確認した結果です。

この結果を受け、色相依存の補正ではなく **C 軸の全方位的な圧縮** を試すことにしました。

---

## 3. C 軸圧縮の効果

最もシンプルな C 軸圧縮は冪乗変換（Power compression）です：

```
C' = C^γ
```

γ=0.675 で COMBVD 全体を最適化すると、STRESS は 47.35 → 32.10 まで改善しました。**1パラメータで STRESS を 32% 削減できる** というのは、C 軸線形性が主要な誤差源であることの傍証です。

なお、この γ=0.675 という値は Luo-Rigg 楕円の真円化最適化（閾値実験）で得られた γ=0.63 と近い値です。独立した実験系と目的関数から求めたパラメータが近い値になるという類似性は興味深いですが、現時点では偶然の一致の可能性も排除できません。

---

## 4. なぜ Naka-Rushton か

冪乗変換の次に検討したのが **Naka-Rushton 関数** です：

```
C' = C^n / (C^n + σ^n)
```

**中 研一（Ken-Ichi Naka）** と **William A.H. Rushton** が 1966 年に発表した研究に基づく関数で、元は網膜の光受容体（錐体・桿体）の光応答モデルです。弱い刺激には敏感に反応し、強い刺激では飽和していく S 字型の特性を持ちます。

冪乗変換との形状上の違いは **飽和特性** にあります：

```
冪乗  C' = C^0.675    → 高彩度域でも C' は単調増加し続ける
NR    C' = C^n/(C^n+σ^n) → C ≫ σ で 1 に収束（飽和）
```

σ パラメータが「感度の切り替わり点」を制御するため、データに応じた適応的なフィッティングが可能です。

---

## 5. 3パラメータで CIEDE2000 を超える

NR 関数による C 軸圧縮に、L 軸の冪乗補正（L' = L^α）を加えた 3パラメータモデルが **Oklch+** です：

```
L' = L^α
C' = C^n / (C^n + σ^n)
h' = h

ΔE = √(ΔL'² + Δa'² + Δb'²)   ← 変換後の Oklab ユークリッド距離
```

L 軸補正単独では効果が薄く（STRESS 40.70）、C 軸の NR 補正と組み合わせたときに効果が現れます。L 補正単独での効果が薄い理由として、C 軸の誤差が L 補正の効果を相殺していると考えられます。両者を同時最適化することでこの相殺が解消されます。

scipy の Nelder-Mead 法で 5 つの初期点から最適化を行い、以下のパラメータに収束しました：

```
α = 0.73,  n = 0.87,  σ = 0.34
COMBVD STRESS = 29.10
```

比較には HELMLAB を含めています。HELMLAB は Oklab を拡張した色差モデルで、COMBVD に 72 パラメータをフィットさせ STRESS 22.48 を達成しています（GitHub: Grkmyldz148/helmlab）。

| モデル | STRESS ↓ | パラメータ数 |
|--------|---------|------------|
| Oklab | 47.35 | 0 |
| Oklch γ=0.675 | 32.10 | 1 |
| **Oklch+（α=0.73, n=0.87, σ=0.34）** | **29.10** | **3** |
| CIEDE2000 | 29.18 | 〜17 |
| HELMLAB v20b | 22.48 | 72 |

**3パラメータで CIEDE2000（〜17定数）を超えました。**

HELMLAB との差（29.10 vs 22.48）は残ります。ただし HELMLAB は COMBVD 3,813ペアに 72パラメータをフィットしており（72/3,813 ≒ 1/53 の比率）、過適合の懸念があります。独立データでの検証は要確認です。

### サブデータセット別スコア

COMBVD は 6 つの独立した実験データセットから構成されています。全サブセットでの改善は汎化の証拠になります。

| モデル | ALL | D65 | M | C | WITT | RIT | LEEDS |
|--------|-----|-----|---|---|------|-----|-------|
| Oklab | 47.35 | 51.45 | 41.69 | 42.22 | 45.01 | 31.76 | 45.15 |
| **Oklch+** | **29.10** | **23.95** | **28.11** | **34.29** | **24.27** | **25.16** | **34.05** |
| Δ | −18.26 | −27.49 | −13.58 | −7.93 | −20.74 | −6.60 | −11.10 |

全 6 サブデータセットで改善しています。改善幅は D65（BFD-P 実験系、全ペアの 53%）で最大（−27.49）、RIT-DuPont（塗料・インク系）で最小（−6.60）です。

ただし RIT-DuPont については注意が必要です。このデータセットの DV（視覚差スコア）を確認すると、全 312ペアで値が 1.02 に完全に一定化されています。DV が定数の場合、STRESS の係数 F の最適化が自明になり、モデルが「色差の大小を正しく予測できるか」ではなく「ΔE のばらつきがどれだけ均一か」を測る状態になります。つまり RIT-DuPont での STRESS は定量的な精度評価として解釈が難しく、改善幅が小さいことの意味も他のデータセットとは異なります。

## 6. 交差検証：汎化の確認

BFD-P(D65) の 2,028ペアをテストセットとして独立させ、残り 1,785ペアで評価した結果です：

```
Train（1,785ペア）: STRESS 33.40
Test（BFD-P D65, 2,028ペア）: STRESS 23.96
```

テストセットで悪化していないことから、過適合はないと判断しています。

## 7. Gamut Mapping — Layer 3

Oklch+ による色差補正（Layer 2）は補間の品質を改善しますが、グラデーション経路上に **sRGB 色域を超えた色が生成される** という実用上の問題があります。

ブラウザや CSS の標準的な処理は、この問題を「事後クリッピング」で対処します。つまり計算結果が色域外に出た後、特定チャンネルを強制的に 0〜1 に切り詰める処理です。この事後処理には問題があります：**クリッピング時に RGB チャンネルの比率が崩れ、輝度階調が不連続に変化する。**

輝度のコントラストは、色覚多様性のあるユーザーにとって色そのものより信頼性の高い情報です。事後クリッピングで輝度構造が壊れると、色相ではなく輝度で情報を読み取るユーザーへの影響が大きくなります。

これに対し、事前に C 軸を sRGB 色域の上限（C_max）以内に制限することで、CSS・ブラウザに渡る前の段階で色域問題を解消できます：

```
Conservative モード: C_max(L)     — 全色相での最小 C 上限（安全な全色相包絡線）
Full モード:        C_max(L, H)   — 色相ごとの正確な C 上限（5° 解像度・2D キャッシュ）
```

**Conservative モード** は全色相に対して安全な単純な L 依存関数です。**Full モード** は色相ごとに精度の高い上限を持ちますが計算コストが高くなります。

Oklch+ 補間（Layer 2）→ Gamut Mapping（Layer 3）という順序で適用することで、知覚的に正しい補間を輝度構造を保ったまま色域内に収めるパイプラインを構成しています。

## 8. 前稿との接続：C 軸パラメータの Hue 変調への転用

前稿「OKLCH 極座標補間の特異点を滑らかに解消する」で使用した Hue 変調の重み関数は：

```javascript
w_h(C) = C^0.87 / (C^0.87 + 0.34^0.87)
```

n=0.87, σ=0.34 という値が、本稿の Oklch+ の C 軸最適化で得られたパラメータです。最適化の順序は **C 軸色差（COMBVD 評価）→ Hue 変調（補間経路の安定化）** であり、色差改善のために決めたパラメータを Hue 変調に転用したものです。

「なぜ Hue 変調にこのパラメータを使うのか」への答えは：**知覚的に均等な彩度スケールで色相への信頼度を制御するから** です。C=σ=0.34 付近で信頼度が 0.5 になる点は、Oklab の彩度スケールにおける知覚的な「中彩度」に対応しており、この点を境に色相の影響を段階的に落とすことが色かぶり解消に効いています。

---

## 9. おわりに

3パラメータで CIEDE2000 を超えるという結果は、Oklab の C 軸が線形距離として扱われているという事実から来ています。Naka-Rushton 関数を選んだ主な理由は飽和特性という関数の形状が彩度の知覚特性と合致していたことです。

残る課題：

- **RIT-DuPont での改善幅が小さい**（−6.60）— このデータセットは全ペアの DV が 1.02 に正規化されており、定量的な精度評価として解釈が困難です。他のデータセットとは性質が異なります
- **L×C 交互作用の補正**（HELMLAB が STRESS 22.48 を達成している要因の一つ）は未実装です
- **自然画像での検証** — 心理物理実験の集計値（COMBVD）での改善が、実際の表示環境でどの程度体感できるかは別途確認が必要です
