---
title: "色を「決める」から「整える」へ — 時彩 1.0 で見えた、AI に渡す前の準備"
---

色は決めたのにAIに「いい感じに作って」と頼むと、なぜか毎回バラつく。

前回の記事で「時彩（ときいろ）」というツールを紹介しました。情景と時間帯を選んで、AI に渡せるカラーパレットを生成する。色決めの迷いを減らす、というのが主題でした。

そこから何度か自分で使ってみて気づいたのは **色が決まってもデザインが揃わない場面がある** ということでした。

理由はわりとシンプルで、**「色」だけ渡しても、AIは "どこに使う色か" を知らない**からです。

---

## "場面を選ぶ" という発想を入れた

時彩 Ver. 1.0 では「用途」という選択肢を追加しました。

- Web UI(日々の出来事)
- マーケティング(初めての印象)
- アクセシビリティ(誰かのために)
- 読み物(静かな余白)

これを選ぶと、**パレット自体の構造が変わります。**

たとえば「マーケティング」を選ぶと、背景にうっすら色みが乗って、primaryが少し鮮やかになる。「読み物」を選ぶと、背景は中性的に落ち着いて、アクセントが控えめになる。「アクセシビリティ」を選ぶと、背景と文字の明暗差が広がる。

同じ「空・海 / 朝」でも、用途が変われば渡るパレットが変わります。

---

## 書体も連動する

用途を選ぶと、書体種別も自動で切り替わります。

| 用途 | 書体種別 | 雰囲気 |
|---|---|---|
| Web UI | ゴシック | 明快・現代的 |
| マーケティング | モダンゴシック | 洗練・都会的 |
| アクセシビリティ | UDゴシック | 読みやすさ特化 |
| 読み物 | ゴシック × 明朝 | 奥行き・余白 |

書体は別途手動で選び直すこともできますが、デフォルトの組み合わせを用途側に持たせておくと、**本文が 創英角ポップ体みたいなチグハグ**が起きにくくなります。(注：おそらく実際には起きにくいです、、、あくまでも例です)

色と書体は別物のようでいて、**「どんな場面のための見た目か」という1点でつながっている**。

---

## ブランドカラーを諦めずに読みやすくする

もう一つ、地味に効いてくると思っているのが「**WCAG AA 保証**」というトグルです。

WCAG は Web のアクセシビリティ基準で、文字と背景のコントラスト比が **4.5:1 以上**あると「読みやすい」とされています。

ここで起きがちなのが、**「ブランドカラーを primary にしたら、コントラスト不足で読みにくい」問題**です。

たとえば「茜・緋」の情景で primary を茜色に選んだ。でも背景の前に置くと、思ったより読みにくい。よくある対処は「じゃあ別の色に変える」です。けれど、それだと **茜色を選んだ意味が消えます。**

時彩 1.0 では、`WCAG AA 保証` トグルを ON にすると、**色の色相と彩度はそのまま、明るさ（L）だけを必要なぶん自動で動かして** 4.5:1 を満たします。

茜色は茜色のまま、ほんの少し明るく（または暗く）なるだけ。**色のアイデンティティを保ったまま、読みやすさを担保できます。**

OKLCH を使っているからこそ可能な設計です。HSL や RGB で同じことをやると、明るさを動かすと色相まで変わってしまうので。

調整が走ったときは「✦ ΔL +0.142 調整済み」のように、どれだけ動いたかが表示されます。コントラスト比も同時に出るので、目で見たときの「読みやすさ」と数値の関係が一画面で確認できます。

---

## 4 軸の選択フローになった

整理すると、Ver. 1.0 の選択は以下の4ステップです。

1. **情景**（色相）— 茜・柿・柳・空・露・藤
2. **時間帯**（明暗）— 深夜・夜明け・朝・正午・夕暮れ・宵
3. **用途**（構造）— Web UI・マーケ・アクセシビリティ(a11y)・読み物
4. **書体種別**（タイポグラフィー）— 用途と連動、手動上書き可

ここに「WCAG AA 保証」トグルを足すと、出来上がるカラーシステムは **AI に渡す前の準備として、ほぼ整っている状態** になります。

書き出した `.md` は前回と同じく Claude ＆ Claude Design/ ChatGPT / Gemini / Tailwind CSS v4 などで使えます。違いは、**色だけでなく書体・サイズまで含まれている** こと。AI に「このカラーシステムで作って」と渡すと、フォントまで合った状態で返ってきます。

---

## 選んだ色を、ツール自体が纏う

地味ですが個人的に気に入っているポイントで、時彩 1.0 では選んだ情景と時間帯に合わせて、**ツール画面自体の外観も変化します**。

具体的には、画面上部から差し込む光、コンテンツ周りの柔らかいグロー。
これらが選んだ色相と明度に連動して、リアルタイムに色味が変わります。

- 「茜・緋 × 夕暮れ」を選ぶと、画面上部に夕焼けのような赤橙の光が差し込む
- 「空・海 × 朝」なら爽やかな青の光
- 「藤・宵」なら紫の落ち着いた光

色を選ぶプロセスそのものが、**ツールの「空気感」** として可視化される仕掛けです。

実用的に言えば、「**この色を選んだら、どんな世界観になるか**」を、AI に渡す前にツール上で先に体感できる、ということでもあります。色の数値だけ見ても掴みにくい「**雰囲気**」が、ツール画面の周辺光として現れるので、選び直しの判断が早くなる気がしています。

---

## 「決めるべきもの」が明示で迷いが減る

色決めの何が大変かというと、**色だけを単独で決められない** ことだと思います。背景が決まっていないと文字色は決められないし、用途が決まっていないと彩度の上げ下げが判断できない。

気づいたのは、**「決めるべきもの」をフローとして明示するだけで、迷いがかなり減る** ということでした。

時彩は「情景 → 時間帯 → 用途 → 書体」の順に決めれば、最後まで迷子にならないように作ってあります。途中で「やっぱり違うかも」となっても、戻って選び直すと全体が連動して動きます。

プロトタイプや個人制作の出発点としてのツール、という位置づけは変えていません。本番のデザインシステムを置き換えるものではなく、**AI 相手に作業するときの "渡す前の整理" を引き受けるツール** です。
